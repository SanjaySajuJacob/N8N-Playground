WRITING QUALITY

Every rewritten or newly added bullet must follow the XYZ accomplishment format:

Action (X) + Measurable Result (Y) + Method/Technology (Z)

Examples:

Reduced deployment time across 4 services by automating CI/CD pipelines with GitHub Actions.

Improved API maintainability across 6 integrations by refactoring Java Spring Boot services.


Metric Estimation Rules

When adding measurable impact:

Prefer factual numbers already present.

Otherwise derive conservative, interview-defensible metrics from existing resume content (modules, APIs, workflows, integrations, projects, years, environments, releases, etc.).

Use scope-based measurements when quantitative business metrics cannot be justified.

Never hallucinate percentages, revenue, customer counts, performance improvements, or scale.

Every metric must be explainable directly from the original resume.

FULL SINGLE-PAGE OPTIMIZATION

The final resume must fill exactly one page: it must NOT overflow to a second page, and it must NOT leave large empty gaps or blank space at the bottom. Aim to fill roughly 90–100% of the usable page height so the page looks complete, balanced, and human-readable. A half-empty page is as much a failure as an overflowing one.

FIRST, judge the current fill level of the base resume relative to a single page.

IF THE PAGE IS UNDERFULL (noticeable empty space, sparse sections, or content ending well above the bottom margin), fill it out by, in this priority order:

1. Lengthening existing bullets — add the concrete method/technology (Z) and measurable scope (Y) that the XYZ format calls for, so short bullets become full, information-dense single lines.

2. Restoring depth to thin entries — give each experience/project a fuller set of bullets (typically 3–5) drawn ONLY from technologies, responsibilities, and achievements already implied by the existing resume. Never invent new employers, projects, dates, or facts.

3. Enriching the Summary and Skills sections with role-relevant keywords and genuinely applicable technologies already supported by the resume.

4. Increasing vertical spacing to distribute content evenly — modestly enlarge inter-section and inter-item spacing (e.g. \vspace, \itemsep, \parskip, or the template's existing spacing commands) so sections breathe and reach the bottom margin, instead of clustering at the top. Only adjust spacing commands that appear AFTER \begin{document}; never touch geometry, margins, or the preamble.

IF THE PAGE IS OVERFULL (content spills past one page):

Rewrite bullets to be concise without losing impact.

Eliminate redundant wording, merge overlapping ideas, and remove filler words.

Reduce whitespace by tightening spacing commands and shortening text — not by deleting valuable accomplishments.

Keep bullets to approximately one rendered line whenever possible.

IN ALL CASES:

Preserve all important accomplishments and maintain readability while maximizing information density.

Prefer real, defensible content over filler: expand with substance, not padding or repetition.

Distribute content and spacing so the page reads as a deliberate, full one-page document.
