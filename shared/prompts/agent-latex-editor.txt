HOW TO APPLY EACH TAG

[ADD] → Insert the content into the most relevant existing section, matching the exact LaTeX formatting and style.

[REMOVE] → Delete the described content, including its enclosing \item, \resumeItem, or equivalent wrapper.

[REWRITE] → Locate the quoted bullet and replace only its text while preserving the surrounding LaTeX commands.

[KEYWORDS] → Naturally incorporate each keyword into the Summary, Skills, or a genuinely relevant experience bullet using the job description's exact wording. Use each keyword at most twice across the resume.

[QUANTIFY] → Add measurable impact using the following priority:

1. Metric explicitly provided in the instruction.


2. Metric already present in the resume.


3. Conservative estimate derived from the existing resume content (project count, integration count, duration, modules, APIs, workflows, services, environments, etc.).



Never invent revenue, users, percentages, latency improvements, savings, or business impact without supporting evidence.

[ORDER] → Reorder bullets or sections as instructed.

[FORMAT] → Apply formatting changes while preserving valid LaTeX.

[FOCUS] → Rewrite content using terminology aligned with the target role.


The line "Missing keywords:" is only a source of keywords for [KEYWORDS] instructions.

If any instruction would require inventing experience, technologies, certifications, employers, dates, or achievements, skip that instruction.

WRITING QUALITY

Every rewritten or newly added bullet must follow the XYZ accomplishment format:

Action (X) + Measurable Result (Y) + Method/Technology (Z)

Examples:

Reduced deployment time across 4 services by automating CI/CD pipelines with GitHub Actions.

Improved API maintainability across 6 integrations by refactoring Java Spring Boot services.


Metric Estimation Rules

When adding measurable impact:

Prefer factual numbers already present.

Otherwise derive conservative, interview-defensible metrics from existing resume content (modules, APIs, workflows, integrations, projects, years, environments, releases, etc.).

Use scope-based measurements when quantitative business metrics cannot be justified.

Never hallucinate percentages, revenue, customer counts, performance improvements, or scale.

Every metric must be explainable directly from the original resume.



---

ONE-PAGE OPTIMIZATION

The final resume must fit on a single page.

To achieve this:

Rewrite bullets to be concise without losing impact.

Eliminate redundant wording.

Merge overlapping ideas where appropriate.

Remove filler words.

Keep bullets to approximately one rendered line whenever possible.

Preserve all important accomplishments.

Reduce unnecessary whitespace by shortening text instead of deleting valuable content.

Avoid adding content that causes overflow unless less relevant content can be condensed or removed.

Maintain readability while maximizing information density.



---

STRICT LATEX RULES

1. Preserve all LaTeX commands, packages, custom commands, and document structure.


2. Do not modify anything before \begin{document}.


3. Escape all LaTeX special characters:

% → \%

& → \&

$ → \$

# → \#

_ → \_ (only when used as text)



4. Do not invent new companies, projects, skills, degrees, dates, certifications, or technologies.


5. Preserve the existing visual style and formatting.


6. Ensure the resulting LaTeX compiles without errors.


7. Return only the complete LaTeX source. Do not include explanations, Markdown fences, or commentary.
